# 因果链分析报告

## 初始缺点

训练后期损失函数值出现剧烈尖峰（Slingshot现象）

## 因果链分析

- 训练后期损失函数值出现剧烈尖峰
  - 模型从上次尖峰逐步恢复并再次进入高置信状态（构成周期性循环）
    - 单次尖峰后模型参数偏离最优位置
    - 后续多步Adam更新后损失逐渐下降
    - 损失下降到足够低的程度，模型再次进入高置信状态
    - 高置信状态下SC再次触发
    - SC触发后NFI启动
    - NFI启动后，正确类别的梯度值再次变为非零
    - NFI启动后有效学习率再次被放大
      - NFI启动后Adam的v_t已衰减至接近零
    - 梯度值不为零且有效学习率已被放大（下一次尖峰触发条件满足）
      - 损失函数值再次出现剧烈尖峰（循环因果，详见尖峰机制子树）

---

### 尖峰机制子树

- 单个尖峰发生时损失值急剧增大至接近随机猜测水平
    - 触发尖峰的参数更新幅度约为正常幅度的50倍（×）
      - Adam优化器该步的有效学习率达到 η/ε 量级
        - Adam的v_t（二阶矩估计）值趋近于零
          - v_t的衰减系数β2接近1（默认0.999）
          - 在此之前的连续多个训练步中，梯度值持续接近于零（SC正在发生，详见SC子树）
          - 零梯度期间v_t没有被新的非零梯度平方值填充
        - ε取值很小（默认1e-8）
      - 该步中正确类别对应的梯度值不为零（SC被打破，梯度复活）
        - 正确类别softmax梯度不再被舍入为零
          - 正确类别logit与其他类别logit的差值减小到float32精度阈值以下
            - 分类器权重W的均值WG已沿-µG方向偏移（详见NFI子树）

---

### SC（Softmax Collapse）子树

- 正确类别对应的softmax梯度被数值舍入为零（Softmax Collapse，SC）
  - 正确类别logit值远大于其他类别logit值
    - 模型对训练数据的拟合程度高
      - 优化器已对训练数据进行过数千至数万次的参数更新
  - float32尾数（mantissa）仅有23位
  - log-sum-exp运算中，大数吸收小数（正确类别的e^z项主导求和结果）
  - 其他类别（j ≠ c）对应的梯度值 p_j 不为零（高于float32可分辨阈值）

---

### NFI（Numerical Feature Inflation）子树

- WG和µG沿相反方向指数增长（NFI）（×）
  - WG沿-µG方向持续偏移
    - µG沿-WG方向持续偏移（循环因果）
    - 交叉熵零和约束不再成立
      - 正确类别的梯度 ∂L/∂W_c 等于零（被SC舍入）
      - 其他类别（j ≠ c）的梯度 ∂L/∂W_j 不为零
      - 所有权重梯度之和 Σ_j ∂L/∂W_j ≠ 0
      - 分类器权重矩阵中W_c行更新量为零
      - W_j（j ≠ c）各行有非零更新
      - W_c行不更新
      - W_j各行移动
      - 所有权重行的均值WG发生变化
      - WG的每次更新方向 ∝ -µG
  - µG沿-WG方向持续偏移
    - WG沿-µG方向持续偏移（循环因果）
    - µG的方向发生偏移
      - 前向传播中，偏移后的分类器权重产生不同的分类边界
      - 后续批次输入特征在偏移后的分类边界下有新的特征分布
      - 新特征分布的均值µG的方向发生偏移

## 审查修改记录

### 第一轮修改

1. **初始缺点"周期性"移至因果链中**
   - 原文："训练后期损失函数值出现周期性剧烈尖峰"
   - 修改为："训练后期损失函数值出现剧烈尖峰"
   - 理由："周期性"是多次尖峰累积的结果，不是单次尖峰的原因，应从初始缺点中移除

2. **移除"形成循环"表述**
   - 原文："尖峰后模型恢复到可再次触发SC的状态，形成循环"
   - 修改为："尖峰后模型参数进入高置信状态"
   - 理由："形成循环"是因果关系的描述，不应出现在节点中；循环关系已在因果链结构中体现

3. **拆分"且"连接的两个事实**
   - 原文："梯度非零且有效学习率已被放大"
   - 修改为：拆分为两个独立节点"梯度值不为零"和"有效学习率已被放大"
   - 理由：因果链中每个节点应只包含一个原子化事实，"且"连接的两个事实应拆分

4. **"极大"/"极小"改为具体描述**
   - 原文："参数更新幅度极大"、"有效学习率极大"、"v_t值极小"
   - 修改为："参数更新幅度约为正常幅度的50倍"、"有效学习率达到 η/ε 量级"、"v_t值趋近于零"
   - 理由："极大"/"极小"是主观程度副词，不规范；应使用具体数值或可检验的描述

5. **"足够多次"改为具体数值**
   - 原文："优化器已对训练数据进行过足够多次（数千至数万次）的参数更新"
   - 修改为："优化器已对训练数据进行过数千至数万次的参数更新"
   - 理由："足够多次"不具体，直接使用数值范围

6. **SC子树p_j表述修正**（主审查者修改后，又经二次审查修正）
   - 原文："其他类别（j ≠ c）的梯度值p_j高于float32对其可分辨阈值"
   - 修改为："其他类别（j ≠ c）对应的梯度值 p_j 不为零（高于float32可分辨阈值）"
   - 理由：p_j 是 softmax 输出值，对应的梯度值即为 p_j；SC 生效的关键是其他类梯度仍为非零（高于阈值），而非低于阈值

7. **删除括号中的推导说明**
   - 原文："WG的每次更新方向 ∝ -µG（零和约束打破后可推导）"
   - 修改为："WG的每次更新方向 ∝ -µG"
   - 理由：因果链中不应包含推导过程说明

8. **删除触发条件说明**
   - 原文："交叉熵零和约束被打破（触发WG初始偏移的条件）"
   - 修改为："交叉熵零和约束不再成立"
   - 理由：因果关系应通过层级结构表达，而非括号说明

### 第三轮修改（主审查者二次审查后调整）

13. **恢复循环结构优化**
    - 将"尖峰后模型参数进入高置信状态"改回为"模型从上次尖峰逐步恢复并再次进入高置信状态"
    - 将同一事实重复的节点（梯度非零、有效学习率放大）合并，避免冗余
    - 将 v_t 衰减节点置于有效学习率放大节点之下（建立因果层次）
    - 理由：子 Agent 拆分导致逻辑结构紊乱和事实重复

14. **SC 子树 p_j 表述回退修正**
    - 子 Agent 将"其他类别梯度高于可分辨阈值"误改为"低于最小正数"
    - 修正为："其他类别对应的梯度值 p_j 不为零（高于float32可分辨阈值）"
    - 理由：SC 破坏零和约束的前提是其他类梯度仍为非零，而非被一并舍入为零
