# 因果链分析报告：训练 Loss 尖峰（Slingshot 机制）

## 初始缺点
训练过程中 loss 出现周期性尖峰（Slingshot 现象），导致训练不稳定、参数异常更新。

## 因果链分析

- 训练过程中 loss 出现周期性尖峰
  - 参数被异常大幅度更新（×）
    - 有效步长极大
      - Adam 二阶矩估计极小
        - 梯度幅度长期处于极低水平
          - 正确类梯度占总梯度的主导部分
          - 正确类梯度消失后，剩余梯度幅值极小
        - 有效学习率达到 η/ε_Adam 量级
          - Adam 更新公式中分母趋近于 ε
            - 二阶矩估计 v̂ 随梯度平方累积趋近于零
            - 分母中的 ε 成为主导项
    - 梯度突然非零
      - 部分训练样本的 logit 跌回到 SC 阈值以下
        - Numerical Feature Inflation（NFI）发生
          - 分类器权重均值 WG 偏移
            - 零和约束被打破
              - 精确算术中 cross-entropy 的类间梯度之和恒为零
              - Softmax Collapse（SC）持续发生
                - 正确类别的 softmax 梯度被精确舍入为零
                  - 正确类别 logit 远大于其他类别 logit
                    - 模型对训练样本的预测置信度提高
                      - 分类器权重和特征表示联合优化
                    - logit 差距超过 float32 精度阈值
                      - float32 的 mantissa 位数有限（23 位）
                      - PyTorch 计算 log-sum-exp 时发生吸收误差
                - 其他类别的 softmax 梯度仍非零
            - 其他类梯度非零且未被抵消
          - WG 沿 µG 反方向漂移
            - 权重更新方向由残差梯度决定
            - 零和约束打破后净梯度方向指向 −µG
          - 特征分布被权重偏移改变
            - 分类器权重直接影响前向传播的特征映射
            - µG 逐渐与 WG 反平行
            - 反向传播将分类器梯度传回特征提取网络
            - 下一轮迭代中特征提取使用更新后的权重
          - 模长指数增长
            - WG 与 µG 互相放大
            - 正反馈回路形成
              - WG 增大导致 µG 向 −WG 偏移
              - µG 偏移加剧 WG 向 −µG 漂移
              - WG 与 µG 模长同步指数增长
      - 参数沿梯度方向一次性大幅位移
    - 训练特征均值 µG 非零
      - 类别不平衡性高（or）
        - 不同类别的样本数量不均
        - 不同类别的特征中心不重合
      - 特征分布有偏
        - 特征提取网络输出分布不对称
        - 无显式约束强制特征均值为零

## 详细子分析：Numerical Feature Inflation（NFI）

- NFI 发生
  - 分类器权重均值 WG 偏移
    - 零和约束被打破
      - 精确算术中 cross-entropy 梯度之和恒为零
      - SC 发生后正确类梯度消失
    - 其他类梯度非零且未被抵消
  - WG 沿 µG 反方向漂移
    - 权重更新方向由残差梯度决定
    - 零和约束打破后净梯度方向指向 −µG
  - 特征分布被权重偏移改变
    - 分类器权重直接影响前向传播的特征映射
    - µG 逐渐与 WG 反平行
    - 反向传播将分类器梯度传回特征提取网络
    - 下一轮迭代中特征提取使用更新后的权重
  - 模长指数增长
    - WG 与 µG 互相放大
    - 正反馈回路形成
      - WG 增大导致 µG 向 −WG 偏移
      - µG 偏移加剧 WG 向 −µG 漂移
      - WG 与 µG 模长同步指数增长
  - 训练特征均值 µG 非零
    - 类别不平衡性高（or）
      - 不同类别的样本数量不均
      - 不同类别的特征中心不重合
    - 特征分布有偏
      - 特征提取网络输出分布不对称
      - 无显式约束强制特征均值为零

## 详细子分析：Softmax Collapse（SC）

- SC 发生
  - 正确类 logit 远大于其他类 logit
    - 模型对训练样本极度确信
    - 正确类概率接近 1
  - float32 精度不足以表示极小梯度
    - 正确类梯度 = p_c − 1 ≈ 0，但在 float32 中被精确舍入为 0
    - 其他类梯度 = p_i（极小但未被舍入为 0）
    - log-sum-exp 计算中发生吸收误差
      - exp 运算中大数吃小数
      - 归一化时小数被忽略
  - 有效学习率达到 η/ε_Adam 量级
    - Adam 更新公式中分母趋近于 ε
      - 二阶矩估计 v̂ 随梯度平方累积趋近于零
      - 分母中的 ε 成为主导项

#### 检查五：解决方案是否对应末端缺点？

文章中提到的解决方案：
1. **float64 计算 logit/loss** —— 对应末端缺点："float32 的 mantissa 位数有限（23 位）"、"PyTorch 计算 log-sum-exp 时发生吸收误差"。✅ 直接对应。
2. **Adam ε 从 1e-8 调到 1e-5** —— 对应末端缺点："Adam 更新公式中分母趋近于 ε"、"有效学习率达到 η/ε_Adam 量级"。提高 ε 可以限制最大有效步长。✅ 直接对应。
3. **classifier 之前加 BatchNorm** —— 对应末端缺点："训练特征均值 µG 非零"、"无 BatchNorm 等约束时 µG 自由漂移"。BN 强制 µG=0，打破 NFI 正反馈。✅ 直接对应。

还有其他可能的解决方案吗？
- **使用 SGD 代替 Adam**：SGD 没有二阶矩累积，不会因梯度长期为零而放大步长。对应末端缺点："Adam 二阶矩估计极小"。这是合理的中间缺点对应的解决方案。
- **梯度裁剪（Gradient Clipping）**：直接限制参数更新幅度。对应"参数更新幅度异常大"。但这只是缓解症状，不解决根本原因。
- **使用混合精度训练的 scale 调整**：调整 loss scale 可能改变梯度幅度，间接影响 SC 阈值。

看起来解决方案都能对应到末端缺点或接近末端的中间缺点。因果链结构合理。
