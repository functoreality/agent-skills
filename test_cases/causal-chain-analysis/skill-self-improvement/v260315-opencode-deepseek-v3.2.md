# 因果链分析报告

## 初始缺点

注意力计算量太大

## 因果链分析

- 注意力计算量太大（×）
  - 注意力计算总计算量大（+）
    - 注意力计算的总操作次数多
      - 注意力计算处理的元素对数量多
        - 注意力计算的输入元素数量多
          - 输入数据总 token 数多（or）
            - 输入文本单词数多
            - 输入图像像素个数多
            - 输入视频帧数多
        - 注意力计算涉及每对输入元素间的关联
      - Transformer 层数多
      - 注意力头数多
      - 批次大小大
    - 每次注意力计算操作平均复杂度高（×）
      - 单次元素对相似度计算复杂度高
        - 查询向量维度高
        - 键向量维度高
        - 点积计算操作存在
        - 缩放除法操作计算开销存在
        - 指数计算操作计算开销大（softmax）
          - 浮点指数函数计算复杂度高
            - 指数函数计算涉及级数展开
            - 指数函数硬件实现电路复杂
      - 注意力权重与值向量加权求和复杂度高
        - 值向量维度高
        - 加权累加操作次数多
          - 每个查询位置与所有键位置计算
      - 注意力计算中的投影操作复杂度高（+）
        - 查询投影计算复杂度高
        - 键投影计算复杂度高  
        - 值投影计算复杂度高
        - 输出投影计算复杂度高
  - 注意力计算中间结果内存占用大（+）
    - 注意力矩阵存储需求大
      - 注意力矩阵元素数量多
        - 注意力矩阵行数多（输入元素数量）
        - 注意力矩阵列数多（输入元素数量）
    - 梯度计算中间结果存储需求大
    - 键值缓存存储需求大（自回归生成时）
      - 缓存序列长度长
      - 缓存维度大
