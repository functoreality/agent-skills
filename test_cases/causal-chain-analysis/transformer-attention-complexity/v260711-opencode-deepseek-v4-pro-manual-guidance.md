# 因果链分析报告：Transformer 注意力计算量

## 初始缺点

注意力计算所需的浮点运算次数过多

## 因果链分析

- 注意力计算所需的浮点运算次数过多：+
	- 查询-键点积所需的浮点运算次数多：×
		- H（注意力头数）大：末端（系统设计决策）
		- L_Q（查询 token 个数）大：×
			- N（总 token 个数）多
				- 待处理数据中的原始信息单元数量多：末端（由应用场景决定，继续分析与本项目无关）
				- 每个 token 覆盖的原始信息单元数量少：末端（系统设计决策：分词/分块粒度）
			- r_Q（每个 token 作为查询的比例）大
				- 注意力采用自注意力模式（每个 token 均产生查询向量）：末端（系统设计决策）
		- L_K（每个查询实际交互的键向量个数）大：×
			- N（总 token 个数）多：上方已分析
			- r_K（每个查询交互的键占全部键的比例）大
				- 注意力计算采用密集交互模式：末端（系统设计决策）
		- d_h（每个头的向量维度）大：末端（系统设计决策）
	- 注意力权重对值向量加权求和所需的浮点运算次数多：×
		- H（注意力头数）大：上方已分析
		- L_Q（查询 token 个数）大：上方已分析
		- L_V（每个查询实际交互的值向量个数）大：×
			- N（总 token 个数）多：上方已分析
			- r_V（每个查询交互的值占全部值的比例）大
				- 注意力计算采用密集交互模式：上方已分析
		- d_h（每个头的向量维度）大：上方已分析
