# 因果链分析报告

## 初始缺点
Transformer 注意力计算量太大

## 因果链分析

- Transformer 注意力计算量太大（×）
  - 注意力机制计算所有位置对之间的相似度
    - 每个查询向量与所有键向量计算点积
    - 相似度计算通过矩阵乘法实现
      - Q 矩阵维度为 [n×d]
      - K^T 矩阵维度为 [d×n]
      - 矩阵乘法计算量为 O(n²d)
  - 序列长度 n 大
    - 输入序列长度大
      - 任务序列长度大
        - 自然语言处理任务文档长
        - 视觉任务图像分辨率高（像素序列长）
        - 音频任务音频片段长
      - 模型上下文窗口大
        - 模型架构最大序列长度设置大
        - 训练数据序列长度分布偏长
  - 特征维度 d 大
    - 模型隐层维度大
      - 模型总参数量大
      - 多头注意力头数量多（总维度 d = 头数 × 每头维度）
    - 投影矩阵 W_q、W_k、W_v 维度大
    - 输出投影矩阵 W_o 维度大
  - 常数因子项大
    - 批处理大小大
    - 注意力头数量多
    - Transformer 层数多（多层堆叠）
  - Softmax 归一化计算复杂度高
    - 每个位置分数进行指数运算
    - 指数运算结果求和并归一化
    - 稳定数值计算引入额外操作
