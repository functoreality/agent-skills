# 因素定义

因果链分析中的每个缺点因素必须是**主语+谓语形式的，明确具体的，原子化的客观事实**。

这一定义贯穿因果链分析全过程：步骤2推演初始缺点、步骤4.1每次展开的即时确认、步骤5检查一收尾全文档复盘。

## 主语+谓语形式

每个因素应采用"主语 + 谓语"的基本句式，表述一个清晰、明确、可检验的事实。

## 明确具体

每个主语、谓语都越具体、越明确越好，不允许模糊泛指。

### 主语精确具体

- 主语应指代具体的实体、属性或度量，避免模糊泛指
- **改进示例**：将`"输入句子长度"大`改为`"输入句子字数"多`，将`"系统响应"慢`改为`"系统响应时间"长`
- **精确区分**：务必区分非常相似、但实际上不同的概念
	- 比如，`参与注意力计算的元素个数`、`输入数据总token数`、`输入文本单词数/图像像素个数/视频帧数`，是三个完全不同的东西
	- 绝不能用笼统的`序列长度`糊弄过去
- **确保专业**：因果链分析对表述精确性的要求相当高。特定领域中经常使用的表述，在因果链分析中可能完全不合适

### 谓语清晰明确

- 谓语可以是形容词（描述状态/性质）或动词（描述动作），可以有宾语
	- 如果涉及比较，可以有比较的对象
- 谓语应具体、可检验，避免抽象的主观评价
- 允许使用"太多"、"效率低"等定性描述
	- 因为它们指向可观测、可比较的属性，能**提示明确的改进方向**
- **禁止使用**："不合理"、"设计不良"等纯价值判断、缺乏明确比较基准的描述
	- 这类表述过于抽象模糊，无法指导具体改进

## 原子化

每个因素应是不可再分的单一事实，不包含隐藏的因果关系或并列关系。

- 一个因素只表述一个事实，不隐含因果关系
	- **错误示例**：`水温高导致气体溶解度低`（包含"导致"关系，应拆分为两个因素：`气体溶解度低` → `水温高`）
- 不合并多个独立因素
	- **错误示例**：`电压存在 AND 物体导电`（应拆分为两个同级因素：`电压存在`和`物体导电`）
- 不将包含多个时序阶段的复合事件表述为单一因素
	- 某些词描述的事件在时间上包含多个先后发生的不同阶段（如"尖峰"包含上升和恢复），将多个不同的事实打包为一个标签
	- 应从复合事件中识别实际构成缺点的阶段，只表述该阶段，而非整个复合事件
	- **错误示例**：`loss出现尖峰`（包含loss上升和loss下降两个阶段，其中只有loss上升是缺点，应改为`loss值上升`）
- 因素只描述根本的偏离事实，不附加冗余信息
	- 冗余信息指去掉后节点仍能表达可识别问题的附加内容，如程度描述（"至接近xxx水平"）、全程共享的范围限定（"（训练后期）"，除非早期状况也需讨论）、不必要的详细展开等
	- **错误示例**：`（训练后期）模型交叉熵损失函数值大幅上升至接近随机猜测水平` → 去掉冗余信息后，`loss上升`已表达可识别的问题

## 客观事实

每个因素必须是可观测、可检验的描述，不依赖于纯粹价值判断或主观动机。

- 不使用"需要xx"、"为了xx"、"旨在xx"等表述动机或目的的短语
	- **错误示例**：`需要提高系统吞吐量`（应改为客观事实：`系统吞吐量低`）
	- **错误示例**：`为了减少延迟而使用了缓存`（只能说客观缺点：`缓存一致性开销大`）

## 肯定陈述

每个因素必须以肯定形式描述实际发生的事实，而非描述"未发生什么"。

- 不使用"未"、"没有"等否定词描述因素本身
- **错误示例**：`纸未保持在白板上的目标位置`（描述的是未发生的事）
- **正确示例**：`纸离开白板上的目标位置`（描述的是实际发生的事）

## 必要时补充状语

若因果链中不同因素节点涉及不同的时间、条件等，不注明会造成混淆，可用括号补充。

- **示例**：`（高峰期）CPU使用率过高` 或 `（打印头回抽时）材料中断`
- **注意**：不能在状语里讨论原因和结果，这些应独立成点以满足原子性

## 检查方法

1. 逐行检查每个缺点因素，判断其是否符合主语+谓语形式
2. 查找以下问题迹象：
	- 包含"导致"、"引起"、"使"等表示因果关系的词汇
	- 包含"和"、"与"、"及"、"或"、"且"等连接多个因素的词汇
	- 包含"要"、"为"、"旨在"、"以"等表示动机的词汇
	- 包含"不合理"、"不良"、"糟糕"等价值判断词汇
	- 包含"未"、"没有"等否定陈述词汇
	- 包含"至"（程度描述）等引入冗余信息的词汇（除非对缺点表述确实必要）
	- **关键建议**：使用 grep 等工具在分析结果文档中搜索相关关键词
		- 也可利用 `scripts/check-notation <file>`
		- 注意，你仍然需要亲自进行基于语义的检查，因为关键词列表无法覆盖所有情况（有误报和漏报）
	- 检查的是逻辑结构而非表面用词。不能用近义词替换（如"使"→"时"、"导致"→"引发"）来规避——若节点在逻辑上仍隐含非规范因素，就应彻底重构
3. 对于不符合要求的因素，拆分或改写以满足上述规范

## 示例对比

| 不符合要求的表述 | 符合要求的表述 |
|----------------|----------------|
| `内存访问不合理` | `内存访问频率过高` |
| `序列长度大` | `参与注意力计算的元素数量多` |
| `系统吞吐量需要提高` | `系统吞吐量低` |
| `水温高导致气体溶解度低` | `气体溶解度低`（原因：`水温高`作为下层因素） |
| `电压存在且物体导电` | `电压存在`<br>`物体导电`（作为两个同级因素） |
| `纸未保持在白板上` | `纸离开白板上的目标位置` |
| `（训练后期）模型交叉熵损失函数值大幅上升至接近随机猜测水平` | `loss上升` |
| `模型参数（分类器权重矩阵W及产生特征向量h的上游网络参数）的数值在单次优化步骤中发生异常大幅度的改变` | `模型参数大幅改变` |

## 技术/数学符号

- 当分析涉及特定领域中通用、专家一看即懂的数学符号时，可用符号表述以提高清晰度
- 推荐格式：符号、含义之一放入括号，如 `L_Q（查询 token 个数）大`，或 `查询 token 个数（L_Q）大`
- 并非所有问题都需要使用数学符号，只有在符号比纯文字更精确、更简洁时才用
