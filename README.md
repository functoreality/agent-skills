## 什么是 Agent Skill？

Agent skill 是一份 Markdown 格式的指令文件，告诉 AI agent 在特定场景下应该如何行动。当用户的问题或意图匹配 skill 的描述时，agent 会自动加载对应的指令，按照预设的规范和流程来完成任务。

如果把 agent 比作一个通用助手，skill 就是给它配备的专业知识手册 —— 不需要每次都从头解释该怎么做。

### 因果链分析

#### 使用场景

你是否遇到过这样的情况：面对一个让你不满的问题，想解决它，却怎么也想不出好办法？
也许你已经尝试过头脑风暴，列出了一些方案，但它们可能要么不切实际，要么治标不治本，要么根本不可行。

因果链分析就是为这种场景设计的。它不直接给你答案，而是先帮你把问题本身彻底搞清楚。

#### 核心思想

通常我们面对问题时，本能的反应是直接去想解决方案。

因果链分析的做法恰恰相反：先花足够的时间**深入分析问题的产生机制**，找到它所依赖的所有潜在的因素。一旦这些依赖因素被清晰而充分地揭示出来，针对它们寻找解决方案就变得直观而系统 —— 这一步其实已经简单非常多了。

#### 能帮你达到什么效果

1. **深度理解问题**：你会获得一份结构化的因果链分析报告，清楚地看到问题的完整产生机制，包括哪些因素是最关键的主干路径。

2. **自然发现解决方案**：分析完成后，你可以直接问 AI："根据你的分析结果，有哪些可能的解决方案？" 此时 AI 给出的回答往往比你一开始凭直觉想到的方案多得多、或许也深刻得多。
你也可以把分析报告导出，交给另一个 AI 或分享给他人讨论，通常能激发出许多之前完全没想到的思路。

3. **超越头脑风暴**：因为分析已经系统性地穷举了所有因果路径，后续的方案搜索不再是漫无边际的"随便想想"，而是有明确靶向的逐一排查。这比纯粹的头脑风暴能覆盖的方案空间大得多。

## 安装

你可以直接把这个仓库告诉你的 AI 智能体，让它来帮你安装。
如果希望亲自安装，可以使用如下命令：

### 安装因果链分析 skill

```bash
npx skills add functoreality/agent-skills/skills/causal-chain-analysis
```

### 安装辅助 skill（可选）

因果链分析生成的报告使用 tab 缩进表示层级关系，推荐安装 outline-read skill，以便使用它的 AI 能够按层级折叠展开、更高效地浏览分析结果：

```bash
npx skills add functoreality/research-notes --skill outline-read
```

## 参考资料

因果链分析的方法论基础来自现代 TRIZ 理论，特别是孙永伟《TRIZ：打开创新之门的金钥匙》一、二两册中介绍的因果链分析方法。

不过，在 AI 辅助执行的具体场景下，本 skill 的分析要求远远超出了书中的抽象指导。书中描述的是一个供人脑执行的框架性方法，而本 skill 面向的是 AI agent，因此需要将每一条分析原则转化为精确、可操作、可验证的规范 —— 包括因素的表述格式、拆分的完备性验证、等价节点的识别与避免重复展开、组合关系的分类（And / Or / + / ×）、末端缺点的判定标准等等。这些大量实操层面的细化，都是在反复使用和迭代中逐渐打磨出来的。
